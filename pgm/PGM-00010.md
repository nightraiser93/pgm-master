# PGM-00010 — Per-project color highlight on ticket cards + review pane

- **Slice value:** at a glance, tell which project a ticket belongs to across the Tasks board and Review queue — today the only signal is a plain-text `projectKey` badge, easy to miss once more than one or two projects are registered.

## Intent
Derive a stable, unique color per `projectKey` — deterministic hash of the key into a fixed
palette (no new config, no manual assignment, survives re-registration) — and use it to
highlight the project badge in:
- [Tasks.tsx](../frontend/src/pages/Tasks.tsx) card header (`TaskCard`, ~L280-303) — the `projectKey` `Badge` at L317-319 (currently only shown when `showProject`).
- [Review.tsx](../frontend/src/pages/Review.tsx) sidebar list item badge (~L214-218) and `DetailPane` header badge (~L274-276).

**Constraint:** cards already use the left border (`border-l-4`) for attention/agent-active
state (amber/blue, [Tasks.tsx:274-282](../frontend/src/pages/Tasks.tsx#L274)) — the project
color must not clobber that existing semantic signal. Tint the project `Badge` itself
(background/border color from the palette) rather than reusing the left-border slot.

## Acceptance criteria
- [ ] Each distinct `projectKey` maps to a consistent color, deterministically (same key -> same color every load, no stored state needed).
- [ ] Color applied to the project badge in Tasks card header, Review sidebar list, and Review detail pane header.
- [ ] Attention (amber) / agent-active (blue) left-border cues on Tasks cards are unaffected.
- [ ] Palette is readable in both light and dark theme (contrast checked, not just default Tailwind shade).

## Subtasks
- [ ] Add a small deterministic hash-to-palette helper (e.g. in `frontend/src/lib/`).
- [ ] Apply it to the project `Badge` in `TaskCard` (Tasks.tsx).
- [ ] Apply it to the project `Badge` in Review.tsx sidebar list + `DetailPane`.
- [ ] Verify contrast/legibility in both themes.
