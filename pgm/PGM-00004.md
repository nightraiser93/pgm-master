# PGM-00004 — pgm-address-comments skill

- **Slice value:** closes the reverse review loop — instead of a human addressing PGM's own PR comments, PGM addresses a human's review comments on its own PR, autonomously where it can, asking where it can't.

## Intent
New invokable skill `pgm-address-comments <TICKET_ID>`. Reads the ticket's `pr:` field
(PGM-00001), fetches unresolved inline review threads via GitHub GraphQL (`reviewThreads` —
REST has no resolved/unresolved field). For each unresolved thread whose latest comment isn't
already `[PGM_BOT]`-prefixed (loop-guard, also skips threads already resolved): analyze the
comment, make a code change + commit if actionable, or reply with a clarifying question if
ambiguous. Every PGM-authored reply is prefixed `[PGM_BOT]`. Replies go out via REST
(`gh api .../pulls/{pr}/comments` with `in_reply_to=<id>`) to stay threaded. Pushes commits to
the existing PR branch directly (no ask-before-push — same auto-push carve-out as
`/pgm-codeit`). Never calls GitHub's resolve-thread mutation — that stays a human action. Full
conversation transcript (file:line, thread state, comment, PGM's reply/action) is written to a
new `## PR Conversations` section in the ticket, updated in place per thread each run — this is
for agent context in future sessions, not meant to be a polished UI (GitHub itself is the UI for
reading the live conversation). Invoked manually by the human whenever they want a pass run —
no auto-discovery, no flag mechanism, no fan-out. Ticket `status:` never changes (stays
`In Review` throughout).

## Acceptance criteria
- [ ] `pgm-address-comments <TICKET_ID>` fetches unresolved review threads for the ticket's `pr:` via GraphQL.
- [ ] Skips threads already resolved, and threads whose latest comment is already `[PGM_BOT]`-prefixed.
- [ ] For actionable comments: makes the code change, commits, pushes to the existing branch, replies "addressed" via REST `in_reply_to`.
- [ ] For ambiguous comments: replies with a clarifying question instead of guessing at a change.
- [ ] `## PR Conversations` section in the ticket is created/updated with the full transcript each run.
- [ ] Never calls a resolve-thread mutation; ticket `status:` never changes.

## Subtasks
- [ ] GraphQL query for `reviewThreads` (isResolved, comments, path, line) via `gh api graphql`.
- [ ] Loop-guard: filter resolved + already-answered-by-PGM_BOT threads.
- [ ] Per-thread: draft code change or clarifying question; commit+push or reply-only.
- [ ] REST reply via `gh api repos/{owner}/{repo}/pulls/{pr}/comments -F in_reply_to=<id>`.
- [ ] Write/update `## PR Conversations` in the ticket body.
