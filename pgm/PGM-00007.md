# PGM-00007 — Add flag/unflag verbs + NEEDS ADDRESS bucket

- **Slice value:** lets a human flag an in-review PR as "needs changes addressed" without inventing a whole new status, and lets tooling discover those tickets in the normal `ready` read.

## Intent
Ticket status stays `In Review` throughout the address-comments loop (no new status value —
decided against it, too much machinery for a marker). Instead add a frontmatter flag
`review_state: changes-requested`. Two new `board.py` verbs:
- `board.py flag <id> [msg]` — human-only, sets the flag (mirrors `approve`/`working` weight).
- `board.py unflag <id> [msg]` — Claude-only, clears it (called at the end of an address-comments pass).

`board.py ready` gains a new "NEEDS ADDRESS" bucket listing every flagged ticket, alongside the
existing READY / WAITING / NEEDS APPROVAL sections, so `/pgm-codeit` (PGM-00009) discovers them
in the same read it already does today.

**Parked** — deferred until PGM-00004 (manual `pgm-address-comments`) is proven useful in
practice; this is the auto-discovery layer on top, not core value. Revisit later.

## Acceptance criteria
- [ ] `board.py flag <id> [msg]` sets `review_state: changes-requested` in frontmatter + stamps a dated `## Comments` entry.
- [ ] `board.py unflag <id> [msg]` clears the field + stamps a comment.
- [ ] `board.py ready` prints a "NEEDS ADDRESS" section listing all tickets with the flag set.
- [ ] Flagging/unflagging never changes `status:` — ticket stays `In Review`.
