# PGM-00001 — Auto-persist PR url on review

- **Slice value:** every ticket knows its own PR without any extra manual step, so downstream tooling (address-comments) can find it.

## Intent
`board.py review <id> "<pr-url>"` already receives a PR url today (used only for the Telegram
ping) but never persists it. Persist it as `pr:` in the ticket's frontmatter, same pattern as
the existing `jira:` field, written inside `cmd_review` the moment a url is passed. This is the
foundation PGM-00004/00005 read from — no new step for the human, `/pgm-codeit`'s existing
`board.py review <id> "<pr-url>"` call starts populating it for free.

## Acceptance criteria
- [ ] `board.py review <id> "<pr-url>"` writes `pr: <url>` into the ticket's frontmatter.
- [ ] Ticket template / board table unaffected when no url is passed (field omitted, not blank).
- [ ] Existing Telegram notify behavior unchanged.

## Subtasks
- [ ] Add `pr:` field write in `cmd_review` (pgm_engine.py), mirroring how `jira:` is set/cleared.
- [ ] Update PGM-SPEC.md ticket frontmatter table to document `pr:`.
