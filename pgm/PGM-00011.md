# PGM-00011 — Rework layout for a dense, modern dashboard feel

- **Slice value:** the dashboard currently reads as a centered marketing page (narrow fixed column, generous whitespace) rather than a data-dense tool — this makes it feel like an analytics/ops dashboard so more tickets/info are visible at once without scrolling.

## Intent
Found the main culprits:
- [Layout.tsx:44](../frontend/src/components/Layout.tsx#L44) + [Layout.tsx:60](../frontend/src/components/Layout.tsx#L60) — both header and `<main>` are `mx-auto max-w-6xl px-6 py-8`, capping content at ~1152px and centering it. On any wide monitor this leaves large empty margins either side. Go fluid (full width with responsive padding, e.g. `px-6 lg:px-10`) or raise the cap substantially (`max-w-[1600px]` or similar) instead of a fixed centered column.
- [Tasks.tsx:197](../frontend/src/pages/Tasks.tsx#L197) — card grid tops out at `xl:grid-cols-4`; with a wider container this should scale further (`2xl:grid-cols-5`/`6`) so density scales with viewport instead of just growing card width.
- Card padding is generous for a dense view — [Tasks.tsx:284](../frontend/src/pages/Tasks.tsx#L284) `CardContent` uses `py-4 gap-3`; tighten padding/gaps across cards, list rows ([Review.tsx:195](../frontend/src/pages/Review.tsx#L195) `px-3 py-2.5`), and page-level `space-y-6` ([Tasks.tsx:123](../frontend/src/pages/Tasks.tsx#L123)).
- Typography: headings (`text-2xl` page titles, [Tasks.tsx:125](../frontend/src/pages/Tasks.tsx#L125)) are oversized for a data tool — scale down heading sizes/weights and rely more on label hierarchy (uppercase micro-labels, tighter line-height) the way analytics dashboards do, rather than editorial-style large headings.
- Colors: current palette ([index.css](../frontend/src/index.css)) is fine as tokens, but panel/card separation is subtle (`--border` at very low chroma) — increase visual separation between panels/rows (e.g. zebra-striping or slightly more contrast on `--card`/`--muted`) so dense grids of data remain scannable.
- Keep both light and dark theme working — this repo already themes via `.dark` class + oklch tokens, don't hardcode colors that break dark mode.

## Acceptance criteria
- [ ] Layout no longer center-constrained at `max-w-6xl` — fluid or a materially wider cap, consistent between header and main.
- [ ] Card grid density scales with viewport width (more columns on wide screens, not just wider cards).
- [ ] Padding/gaps tightened across cards, list rows, and page sections — less dead whitespace, same information readable.
- [ ] Heading/label typography rescaled to read as a dashboard, not a marketing page (smaller headings, clearer micro-labels).
- [ ] Panel/row/card boundaries more visually distinct at a glance (contrast increase, not just border color already there).
- [ ] Both light and dark themes still look correct — no regressions in either.

## Subtasks
- [ ] Widen/de-center the `Layout.tsx` shell (header + main).
- [ ] Increase grid density breakpoints in Tasks.tsx (and Review.tsx list panel width/columns if applicable).
- [ ] Tighten padding/gap utility classes across Tasks.tsx, Review.tsx cards/rows.
- [ ] Rescale heading/label type styles.
- [ ] Tune `--border`/`--card`/`--muted` contrast (or add a subtle stripe/hover treatment) for scannability.
- [ ] Manually check both themes after changes.
