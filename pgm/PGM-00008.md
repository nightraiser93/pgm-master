# PGM-00008 — UI Flag button

- **Slice value:** human can mark "changes requested" from the dashboard, same click-to-act ergonomics as the existing Approve/Working gate buttons — no CLI needed for the one human-triggered step in the loop.

## Intent
`app.py`/`frontend` already render Approve and Working as clickable buttons that shell out to
the project's own `board.py` via `/api/action`. Add a third button, "Flag", visible on
`In Review` tickets, that calls `board.py flag <id>`. `unflag` stays code-only (only the
address-comments flow calls it) — no UI needed for that direction.

**Parked** — depends on PGM-00007 which is itself deferred; revisit together.

## Acceptance criteria
- [ ] `In Review` tickets show a "Flag" action button in the dashboard.
- [ ] Clicking it calls `/api/action` -> `board.py flag <id>` for that ticket's project.
- [ ] Flagged tickets visibly show their `review_state: changes-requested` (e.g. a badge), consistent with existing status badges.
- [ ] No "Unflag" button added anywhere in the UI.

## Subtasks
- [ ] Extend `/api/action` handling in app.py to accept `flag`.
- [ ] Add Flag button + badge to Tasks.tsx / Review.tsx, matching existing Approve/Working button styling.
